#EJERCICIO 1
Para este ejercicio decidí usar una clase abstracta llamada Figura para tener una base común para todas las figuras y no repetir código. Ahí definí los atributos principales y los métodos abstractos para el área y el perímetro, así obligo a que cada figura calcule esos valores a su manera. También hice un método describir() para mostrar toda la información sin tener que repetirlo en cada clase. Luego creé las clases Circulo, Rectangulo y TrianguloRectangulo que heredan de Figura y sobrescriben los métodos según sus fórmulas. Finalmente, usé un arreglo de tipo Figura para poder manejar todas las figuras juntas y aplicar polimorfismo.

#EJERCICIO 2
En este caso usé una clase abstracta Empleado para definir lo básico que todos los empleados tienen, como nombre, id y horas trabajadas. A partir de ahí creé diferentes tipos de empleados según su forma de pago, como por hora, fijo y por comisión, cada uno con su propia lógica para calcular el salario. También utilicé una interfaz Bonificable porque no todos los empleados reciben bono, entonces solo algunas clases la implementan. Para identificar cuáles empleados tienen bono usé instanceof, lo que me permite aplicar esa funcionalidad solo cuando corresponde. En general, este diseño me ayuda a tener el código más organizado y fácil de extender si se agregan nuevos tipos de empleados.
